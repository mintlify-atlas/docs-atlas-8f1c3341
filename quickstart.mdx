---
title: "Quick Start"
description: "Get from zero to your first file transfer in under two minutes."
---

Follow these steps to install WebTransfer Lite, configure it, and transfer your first file.

<Steps>
  <Step title="Clone or download the project">
    Clone the repository to your machine:

    ```bash
    git clone https://github.com/Michinded/webtransfer-lite.git
    cd webtransfer-lite
    ```

    Or download the source archive from GitHub and extract it.
  </Step>

  <Step title="Install dependencies">
    Install all required packages using pnpm:

    ```bash
    pnpm install
    ```

    <Note>
      pnpm and Node.js 18 or later are required. If you don't have pnpm, install it with `npm install -g pnpm`.
    </Note>
  </Step>

  <Step title="Initialize the environment">
    Generate your `.env` file with a secure, randomly generated `JWT_SECRET`:

    ```bash
    pnpm run init
    ```

    This copies `.env.example` to `.env` and fills in a 128-character hex secret. You only need to run this once.

    The resulting `.env` file looks like this:

    ```env
    JWT_SECRET=<generated secret>
    PORT=3000
    UPLOADS_DIR=uploads
    SESSION_HOURS=2
    MAX_FILE_SIZE_MB=0
    ```

    You can edit `PORT`, `UPLOADS_DIR`, `SESSION_HOURS`, and `MAX_FILE_SIZE_MB` to suit your needs. Do not change `JWT_SECRET` — if you need a new one, delete `.env` and run `pnpm run init` again.

    <Tip>
      Set `MAX_FILE_SIZE_MB` to a positive number to cap individual upload sizes and protect disk space. Leave it at `0` for no limit — files are always streamed to disk regardless, so large transfers won't exhaust RAM.
    </Tip>
  </Step>

  <Step title="(Optional) Validate the environment">
    Before starting the server, confirm that your `.env` values are valid:

    ```bash
    pnpm run check
    ```

    The check script verifies that `JWT_SECRET` is set and long enough, `PORT` is a valid integer between 1024 and 65535, `SESSION_HOURS` is at least 0.5, `MAX_FILE_SIZE_MB` is a non-negative number, and `UPLOADS_DIR` is writable (creating it if it doesn't exist).

    A passing run looks like:

    ```
      ✓  .env found
      ✓  JWT_SECRET set (128 chars)
      ✓  PORT=3000
      ✓  SESSION_HOURS=2
      !  MAX_FILE_SIZE_MB=0 — no file size limit enforced
      ✓  UPLOADS_DIR writable (/path/to/webtransfer-lite/uploads)

      All checks passed. Ready to run pnpm start.
    ```

    Fix any errors reported before proceeding.
  </Step>

  <Step title="Start the server">
    Start WebTransfer Lite:

    ```bash
    pnpm start
    ```

    After a moment, the terminal displays your connection details and a QR code:

    ```
    ──────────────────────────────────
       WebTransfer Lite
    ──────────────────────────────────
      URL      : http://192.168.1.x:3000
      Password : AB3X7K
      Session  : 2h
    ──────────────────────────────────

      Scan to connect (QR = instant access):
      [ QR code ]
    ```

    The server is now live on your local network. The session lasts for the duration configured in `SESSION_HOURS` (default: 2 hours). When the session expires, the server shuts down automatically and frees the port.

    <Warning>
      Each `pnpm start` generates a new random password. Share only the URL and password with people on your local network whom you trust.
    </Warning>
  </Step>

  <Step title="Connect from your device">
    **From a PC on the same network:**
    1. Open the URL shown in the terminal (e.g., `http://192.168.1.x:3000`) in any browser.
    2. Enter the password displayed in the terminal.
    3. You now have access to the file transfer interface.

    **From a mobile device:**
    1. Open your phone's camera or a QR scanner app.
    2. Point it at the QR code in the terminal.
    3. Tap the link. The pre-signed token in the URL grants instant access — no password needed.

    <Tip>
      The QR code contains a pre-signed JWT, so scanning it is faster than entering the password manually. Use it whenever you're connecting from a phone or tablet.
    </Tip>
  </Step>

  <Step title="Transfer your first file">
    In the browser interface:

    - **Upload:** drag and drop files onto the upload area, or click to select them. Files land in the `UPLOADS_DIR` directory on the server machine.
    - **Download:** click any listed file to download it to your device.
    - **Delete:** remove files you no longer need from the interface.

    If you upload a file whose name already exists, the server automatically renames the new file with a timestamp suffix — for example, `photo.png` becomes `photo-1712345678901.png` — so no existing file is overwritten.
  </Step>
</Steps>

## Development mode

If you are modifying the server code, use `pnpm run dev` instead of `pnpm start`:

```bash
pnpm run dev
```

This starts the server with `nodemon`, which automatically restarts it whenever you save a file. Not intended for production use.

## What happens when the session expires

When `SESSION_HOURS` elapses, the server prints a message and exits:

```
  Session expired. Server shutting down.
```

The port is freed immediately. To start a new session, run `pnpm start` again. A new password and QR code are generated each time.
