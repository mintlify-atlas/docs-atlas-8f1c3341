title: "Environment variables"
description: "Configure WebTransfer Lite using a .env file. Learn what each variable controls, its valid range, and the defaults."
---

All configuration is done through a `.env` file in the project root. Run `pnpm run init` to generate this file from `.env.example`. Do not create or edit `.env` manually before running init — `JWT_SECRET` must be generated by the init script.

## .env.example

The template shipped with the project:

```env .env.example
JWT_SECRET=
PORT=3000
UPLOADS_DIR=uploads
SESSION_HOURS=2
MAX_FILE_SIZE_MB=0
```

## Variables

<ParamField path="JWT_SECRET" type="string" required>
  A 128-character hex string used to sign and verify JWT session tokens. This value is generated automatically by `pnpm run init` (128 random bytes encoded as hex via `crypto.randomBytes(64).toString('hex')`).

  **Do not set this manually.** If you need a new secret, delete `.env` and run `pnpm run init` again. The server exits with an error if this value is missing or empty.
</ParamField>

<ParamField path="PORT" type="number" default="3000">
  The TCP port the server listens on. Must be an integer between `1024` and `65535` (inclusive). Ports below 1024 require elevated privileges on Linux and macOS and are rejected by the check script.

  ```env
  PORT=8080
  ```
</ParamField>

<ParamField path="SESSION_HOURS" type="number" default="2">
  How long each server session lasts, in hours. Must be a number greater than or equal to `0.5`. When the session duration elapses, the server prints a message and exits automatically, freeing the port.

  Values above `24` are accepted but trigger a warning from `pnpm run check`.

  ```env
  SESSION_HOURS=4
  ```
</ParamField>

<ParamField path="MAX_FILE_SIZE_MB" type="number" default="0">
  Maximum size of a single uploaded file, in megabytes. Set to `0` to enforce no limit (the default).

  When a limit is set, the server responds with HTTP `413` if an upload exceeds it. Files are always streamed directly to disk regardless of size, so large uploads won't exhaust server RAM even when no limit is configured.

  <Warning>
    `MAX_FILE_SIZE_MB=0` means **no file size limit**. Any file, regardless of size, will be accepted. Set a positive value to protect disk space on the host machine.
  </Warning>

  ```env
  MAX_FILE_SIZE_MB=500
  ```
</ParamField>

<ParamField path="UPLOADS_DIR" type="string" default="uploads">
  Path to the directory where uploaded files are stored. Accepts absolute and relative paths. Relative paths are resolved from the project root using `path.resolve()`.

  The server creates this directory automatically on startup if it does not already exist. See [Upload directory](/configuration/uploads) for examples and platform-specific guidance.

  ```env
  UPLOADS_DIR=/home/user/transfers
  ```
</ParamField>

## Validating your configuration

Run the check script before starting the server to catch configuration errors early:

```bash
pnpm run check
```

The script validates every variable and reports errors, warnings, and passing checks:

```
  WebTransfer Lite — environment check

  ✓  .env found
  ✓  JWT_SECRET set (128 chars)
  ✓  PORT=3000
  ✓  SESSION_HOURS=2
  !  MAX_FILE_SIZE_MB=0 — no file size limit enforced
  ✓  UPLOADS_DIR writable (/home/user/webtransfer-lite/uploads)

  All checks passed. Ready to run pnpm start.
```

<Tip>
  Run `pnpm run check` any time you edit `.env`. It will surface invalid values, unreachable directories, and missing secrets before they cause a runtime failure.
</Tip>
