title: "Security"
description: "How WebTransfer Lite authenticates clients, protects files, and limits exposure."
---

WebTransfer Lite is designed for short-lived, LAN-only file transfers. Its security model is intentionally simple and requires no user accounts or persistent configuration beyond a one-time setup.

## Password authentication

Each time the server starts, a new 6-character password is generated by `generatePassword()` using `crypto.randomBytes`. The character set is:

```
ABCDEFGHJKLMNPQRSTUVWXYZ23456789
```

Visually ambiguous characters — `0`, `O`, `1`, and `I` — are excluded to prevent misreading on small screens or printed output.

The password is held in memory only (`global.serverPassword`) and is never written to disk or logged to a file. When the session ends and the server exits, the password is gone.

To obtain a JWT, a client POSTs the password to `POST /api/auth`:

```bash
curl -X POST http://192.168.1.x:3000/api/auth \
  -H "Content-Type: application/json" \
  -d '{"password": "AB3X7K"}'
```

A correct password returns a signed token:

```json
{ "token": "<jwt>" }
```

An incorrect password returns `401`:

```json
{ "error": "Invalid password" }
```

## JWT tokens

All file API endpoints (`/api/files/*`) require a valid JWT. The middleware accepts the token in two forms:

- **Authorization header** (preferred for API calls): `Authorization: Bearer <token>`
- **Query parameter** (used for direct download links): `?token=<token>`

Tokens are signed with the `JWT_SECRET` from your `.env` file and carry the payload `{ authorized: true }`. They expire at the same time as the server session — `SESSION_HOURS` hours after the server started (minimum 0.5 hours).

The QR code displayed in the terminal embeds a pre-signed token in the URL. Scanning it logs the user in immediately without requiring a password. That token has the same expiry as any other token issued during the session.

Invalid or expired tokens return `401`:

| Condition | Error message |
|---|---|
| Token missing | `Unauthorized` |
| Token expired | `Session expired` |
| Token invalid | `Invalid token` |

When a client receives a `401`, it is automatically logged out and redirected to the login screen.

## Session auto-expiry

The server uses a `setTimeout` set to `SESSION_HOURS * 60 * 60 * 1000` milliseconds. When it fires:

1. The server prints `Session expired. Server shutting down.`
2. `process.exit(0)` is called.
3. The port is freed and the machine is no longer exposed on that port.

The minimum session duration is 0.5 hours (30 minutes), enforced by `Math.max(0.5, ...)` in the startup code. Setting `SESSION_HOURS` below `0.5` in `.env` is silently clamped to `0.5`.

## LAN-only exposure

The server binds to `0.0.0.0`, which means it accepts connections on all network interfaces on the configured port. It is accessible to any device on the same network — but only that network.

<Warning>
  WebTransfer Lite is **not** protected against the open internet unless you deliberately configure port forwarding on your router. Do not use it on untrusted public Wi-Fi networks (cafes, airports, hotels), where other devices on the same network could connect to the server and access your files.
</Warning>

## Best practices

<CardGroup cols={2}>
  <Card title="Keep sessions short" icon="clock">
    Use a low `SESSION_HOURS` value for sensitive transfers. The server shuts down automatically when the session ends, limiting the window of exposure.
  </Card>
  <Card title="Guard the terminal output" icon="eye-slash">
    The URL, password, and QR code together grant full access. Do not share screenshots of the terminal or leave it visible on a shared screen.
  </Card>
  <Card title="Set a file size limit" icon="hard-drive">
    Configure `MAX_FILE_SIZE_MB` to a reasonable value. Without a limit, any authenticated client can fill your disk. Files are always streamed to disk, so the limit does not affect memory usage.
  </Card>
  <Card title="Use a trusted network" icon="wifi">
    Run the server only on private, trusted networks. Avoid public Wi-Fi where other unknown devices share the same network segment.
  </Card>
</CardGroup>
